What are LLMs vs SLMs?

LLMs (Large Language Models)

Massive cloud-based neural networks (typically 30B to 1000B+ parameters). Extremely smart, high reasoning capacity, but require high-end GPU servers and paid API queries.

SLMs (Small Language Models)

Compact, highly optimized models (typically 1B to 9B parameters). Capable of running locally on consumer smartphones, web browsers, or laptops. Ideal for privacy and offline usage.

The Lifecycle of a Custom Language Model

How a model transitions from raw text into a smart AI on your phone

1

1. Pre-Training (Self-Supervised)

A huge neural network reads trillions of words of web content. Its sole task is to predict the next word. It gains general grammar, world facts, and language skills, but cannot follow instructions yet. This costs millions of dollars in compute.

PyTorchTensorFlow1000s of H100 GPUs
2

2. Fine-Tuning (SFT - Supervised Fine-Tuning)

You train the pre-trained base model on clean, curated instructions. For example, prompt-response pairs: {"prompt": "Translate 'hello' to Spanish", "response": "Hola"}. This teaches the model to listen and act as an assistant.

LLaMA-FactoryUnslothHugging Face Transformers
3

3. Alignment (RLHF / DPO)

Human annotators rate model outputs. Direct Preference Optimization (DPO) or Reinforcement Learning from Human Feedback (RLHF) updates model weights so that outputs align with safety, tone, helpfulness, and style standards.

TRL LibraryRLHF Annotators
4

4. Quantization (Compression)

Raw model weights are in 16-bit floats (FP16), which is too large for phones. Quantization maps these float values to smaller 4-bit or 8-bit integers (INT4/INT8). A 7B parameter model shrinks from 14GB down to 3.8GB, fitting on a phone with minor loss in accuracy.

Create an instruction dataset in JSONL format. Each line represents a training sample with system instructions, user prompts, and expected outputs.

{"messages": [{"role": "system", "content": "You are a helpful customer assistant."}, {"role": "user", "content": "Track order #104"}, {"role": "assistant", "content": "Your order #104 has been shipped and is scheduled for delivery tomorrow..."}]}

Use the Unsloth framework in a Google Colab notebook for extremely fast, low-memory LoRA fine-tuning:

train.py (Python) 
from unsloth import FastLanguageModel
import torch

# Load model fast!
model, tokenizer = FastLanguageModel.from_pretrained(
    model_name = "unsloth/gemma-2-2b-it",
    max_seq_length = 2048,
    load_in_4bit = True, # QLoRA
)

# Set up LoRA Adaptors
model = FastLanguageModel.get_peft_model(
    model,
    r = 16, # Rank
    target_modules = ["q_proj", "k_proj", "v_proj", "o_proj", "gate_proj", "up_proj", "down_proj"],
    lora_alpha = 16,
    lora_dropout = 0,
)

Convert your trained weights into GGUF format (used by llama.cpp and mobile loaders) and compress to 4-bit:

Terminal (using llama.cpp) 
# 1. Install llama.cpp dependencies
git clone https://github.com/ggerganov/llama.cpp
cd llama.cpp && make

# 2. Convert PyTorch model to unquantized GGUF
python3 convert_hf_to_gguf.py /path/to/my_fine_tuned_model --outfile model-f16.gguf

# 3. Quantize weights to 4-bit (Q4_K_M)
./llama-quantize model-f16.gguf model-q4_k_m.gguf Q4_K_M

Flutter & Dart Integration Guide

Complete developer pathways for integrating AI capabilities into Flutter applications

Package: google_generative_ai
Size Cost: < 500 KB (Zero weight model)
OS Support: iOS, Android, Web, Desktop
Memory usage: Negligible (Standard HTTP calls)

This approach issues secure network requests directly to Google Gemini APIs. Ideal when you require maximum reasoning capacity, multi-modality, and minimal app bundle size overhead.

pubspec.yaml 
dependencies:
  flutter:
    sdk: flutter
  google_generative_ai: ^0.4.0
gemini_service.dart 
import 'package:google_generative_ai/google_generative_ai.dart';

class GeminiService {
  final GenerativeModel _model;

  GeminiService(String apiKey) 
      : _model = GenerativeModel(
          model: 'gemini-1.5-flash', // fast & low cost model
          apiKey: apiKey,
          // optional system instruction to guide model personality
          systemInstruction: Content.system("You are a helpful assistant for Flutter developers."),
        );

  Future askModel(String query) async {
    try {
      final response = await _model.generateContent([Content.text(query)]);
      return response.text ?? "Error: Empty response";
    } catch (e) {
      return "Network Exception: $e";
    }
  }
}
SDK / Tool: Google MediaPipe Tasks (LLM Inference)
Model format: .bin (MediaPipe quantized Gemma/Llama)
RAM usage: ~1.4 GB to 2.2 GB
App Size Overhead: ~40 MB (native C++ libraries)

Google's MediaPipe provides cross-platform hardware-accelerated bindings (GPU via Metal/Vulkan) to run SLMs directly on iOS and Android devices, guaranteeing low-latency response offline.

pubspec.yaml 
dependencies:
  flutter:
    sdk: flutter
  # Uses Google's official mediapipe genai tasks
  mediapipe_genai: ^0.1.0
mediapipe_service.dart 
import 'package:mediapipe_genai/mediapipe_genai.dart';

class MediaPipeService {
  LlmInference? _engine;

  Future initModel(String modelPath) async {
    final options = LlmInferenceOptions(
      modelPath: modelPath, // Path to local .bin file (e.g. gemma-2b-it-cpu.bin)
      maxTokens: 512,
      temperature: 0.7,
    );
    _engine = await LlmInference.create(options);
  }

  Future generate(String prompt) async {
    if (_engine == null) return "Model not initialized!";
    
    // Executes 100% offline, on-device CPU/GPU
    final response = await _engine!.generateResponse(prompt);
    return response;
  }
  
  void dispose() {
    _engine?.close();
  }
}
Framework: llama.cpp native bindings via Dart FFI
Model format: .gguf (INT4, INT8, FP16)
RAM usage: Scalable (e.g., 800MB - 8GB)
Customizability: Extremely high (custom quantization, custom kernels)

Integrate llama.cpp using Dart Foreign Function Interface (FFI). This gives maximum control over CPU thread count, GPU layer offloading (on macOS/iOS), and allows loading any standard GGUF model directly from the mobile file storage.

pubspec.yaml 
dependencies:
  flutter:
    sdk: flutter
  # Dart bindings for llama.cpp (llama_cpp_dart or custom FFI wrapper)
  llama_cpp_dart: ^0.2.1 
  path_provider: ^2.1.1 # to locate GGUF models on mobile storage
llama_service.dart 
import 'package:llama_cpp_dart/llama_cpp_dart.dart';
import 'package:path_provider/path_provider.dart';

class LlamaLocalService {
  LlamaModel? _model;
  LlamaContext? _context;

  Future loadModel(String filename) async {
    final docDir = await getApplicationDocumentsDirectory();
    final modelPath = "${docDir.path}/$filename"; // e.g. gemma-2b-q4_k_m.gguf

    // Set parameters
    final params = ModelParams()
      ..useGpu = true // offload layers to Apple Metal (CoreML) or Vulkan
      ..gpuLayers = 16; 

    _model = LlamaModel(modelPath, params);
    
    final contextParams = ContextParams()..contextSize = 2048;
    _context = LlamaContext(_model!, contextParams);
  }

  String chat(String prompt) {
    if (_context == null) return "Model not loaded";
    
    // Feed tokens to context and run local evaluation loop
    final tokens = _context!.tokenize(prompt, addBos: true);
    _context!.eval(tokens);
    
    // Retrieve output tokens
    final response = _context!.generateResponse();
    return response;
  }
}
